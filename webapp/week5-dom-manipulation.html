What is the DOM?

DOM stands for Document Object Model. It's how JavaScript sees and interacts with your HTML page.

Think of it like this:
‚Ä¢ Your HTML is a document (a web page)
‚Ä¢ The browser turns it into objects JavaScript can work with
‚Ä¢ JavaScript can then read and change these objects
‚Ä¢ Changes happen instantly on the page!

The DOM is the bridge between HTML and JavaScript.

The DOM Tree

The browser sees your HTML as a tree structure:

<html> <head> <title>My Page</title> </head> <body> <h1 id="title">Hello</h1> <p class="text">Content</p> </body> </html>

JavaScript can access each element like objects in a tree:

document

      ‚îú‚îÄ head
      ‚îÇ   ‚îî‚îÄ title
      ‚îî‚îÄ body
          ‚îú‚îÄ h1 (id="title")
          ‚îî‚îÄ p (class="text")

Why DOM Manipulation Matters

Everything interactive on a website uses DOM manipulation:

  • Facebook: When you click "Like", DOM changes the button color
  • Gmail: When you delete an email, DOM removes it from the list
  • Netflix: When you hover over a movie, DOM shows the description
  • Shopping carts: When you add an item, DOM updates the count

The document Object

The document object represents your entire HTML page. It's your starting point for all DOM manipulation:

// The document object is always available console.log(document); // Shows the entire HTML console.log(document.title); // Shows page title console.log(document.body); // Shows the body element
üí° Important: JavaScript can only access the DOM after the HTML has loaded. That's why we put <script> tags at the end of the <body>, or use event listeners like DOMContentLoaded.

Selecting Elements

Before you can change an element, you need to select (find) it. There are several ways to do this:

1. Select by ID

Use getElementById() to find an element with a specific ID:

// HTML: <h1 id="title">Hello</h1> let title = document.getElementById('title'); console.log(title); // Shows: <h1 id="title">Hello</h1>
üí° Remember: IDs should be unique - only one element should have each ID. Use getElementById (no 's' at the end) because you're getting one element.

2. Select by Class

Use getElementsByClassName() to find all elements with a class:

// HTML: <p class="text">Para 1</p> // <p class="text">Para 2</p> let paragraphs = document.getElementsByClassName('text'); console.log(paragraphs); // HTMLCollection with both <p> elements console.log(paragraphs[0]); // First paragraph console.log(paragraphs[1]); // Second paragraph

3. Select by Tag Name

Use getElementsByTagName() to find all elements of a type:

let allParagraphs = document.getElementsByTagName('p'); let allDivs = document.getElementsByTagName('div'); let allButtons = document.getElementsByTagName('button');

4. Query Selector (Modern Way!) ‚≠ê

Use querySelector() with CSS selectors - the most flexible method:

// Get first matching element let title = document.querySelector('#title'); // By ID let para = document.querySelector('.text'); // By class let button = document.querySelector('button'); // By tag // Get ALL matching elements let allParas = document.querySelectorAll('.text'); let allButtons = document.querySelectorAll('button'); // Complex selectors (just like CSS!) let firstPara = document.querySelector('div p'); // p inside div let activeBtn = document.querySelector('.btn.active'); // Multiple classes
querySelector vs querySelectorAll:
‚Ä¢ querySelector() - Returns FIRST matching element
‚Ä¢ querySelectorAll() - Returns ALL matching elements (NodeList)

Use querySelectorAll() when you want to work with multiple elements!

Comparison Table

Method Returns Example
getElementById Single element ('title')
getElementsByClassName HTMLCollection ('text')
querySelector Single element ('#title')
querySelectorAll NodeList ('.text')
üí° Best Practice: Use querySelector() and querySelectorAll() for most cases - they're more flexible and use CSS selectors you already know!

Checking if Element Exists

let element = document.querySelector('#myId'); if (element) { console.log("Element found!"); } else { console.log("Element doesn't exist"); }
Common Mistakes:
‚ùå document.getElementById('#title') (no # needed!)
‚úÖ document.getElementById('title')

‚ùå document.querySelector('title') (missing #)
‚úÖ document.querySelector('#title')

Manipulating Elements

Once you've selected an element, you can change it in many ways!

1. Changing Text Content

1 textContent - Get or set text (no HTML)
// HTML: <h1 id="title">Old Title</h1> let title = document.querySelector('#title'); // Get text console.log(title.textContent); // "Old Title" // Set text title.textContent = "New Title"; // Changes to "New Title"
2 innerHTML - Get or set HTML content
let div = document.querySelector('.content'); // Get HTML console.log(div.innerHTML); // Set HTML (can include tags!) div.innerHTML = '<p>New <strong>paragraph</strong></p>';
üí° textContent vs innerHTML:
‚Ä¢ Use textContent for plain text (safer)
‚Ä¢ Use innerHTML when you need to add HTML tags

2. Changing Styles

Modify CSS directly from JavaScript:

let box = document.querySelector('.box'); // Change individual styles box.style.color = 'red'; box.style.backgroundColor = 'yellow'; // Note: backgroundColor not background-color box.style.fontSize = '24px'; box.style.border = '2px solid blue'; box.style.padding = '20px'; box.style.display = 'none'; // Hide element box.style.display = 'block'; // Show element
CSS Property Name Changes:
CSS dashes become camelCase in JavaScript:
‚Ä¢ background-color ‚Üí backgroundColor
‚Ä¢ font-size ‚Üí fontSize
‚Ä¢ border-radius ‚Üí borderRadius

3. Working with Classes

Instead of changing individual styles, it's better to add/remove CSS classes:

let button = document.querySelector('.btn'); // Add a class button.classList.add('active'); // Remove a class button.classList.remove('active'); // Toggle a class (add if not there, remove if there) button.classList.toggle('active'); // Check if class exists if (button.classList.contains('active')) { console.log("Button is active!"); }
üí° Best Practice: Use classList instead of modifying style directly. Define styles in CSS, then add/remove classes with JavaScript. This keeps styling in CSS where it belongs!

4. Changing Attributes

Modify HTML attributes like src, href, alt:

let img = document.querySelector('img'); // Get attribute let source = img.getAttribute('src'); console.log(source); // Set attribute img.setAttribute('src', 'new-image.jpg'); img.setAttribute('alt', 'New description'); // Remove attribute img.removeAttribute('alt'); // Shortcut for common attributes img.src = 'another-image.jpg'; img.alt = 'Another description';

5. Getting/Setting Values

For form elements like inputs:

let input = document.querySelector('#username'); // Get value let value = input.value; console.log(value); // Set value input.value = "John Doe"; // Clear value input.value = "";

Handling Events

Events are things that happen on the page: clicks, typing, mouse movement, scrolling, etc. JavaScript can "listen" for these events and respond!

What Are Events?

Common Events:
‚Ä¢ click - User clicks on element
‚Ä¢ dblclick - User double-clicks
‚Ä¢ mouseenter - Mouse enters element
‚Ä¢ mouseleave - Mouse leaves element
‚Ä¢ keydown - User presses a key
‚Ä¢ keyup - User releases a key
‚Ä¢ input - Input value changes
‚Ä¢ submit - Form is submitted
‚Ä¢ load - Page finishes loading

Adding Event Listeners

Use addEventListener() to listen for events:

// HTML: <button id="myBtn">Click Me</button> let button = document.querySelector('#myBtn'); // Listen for click event button.addEventListener('click', function() { console.log("Button was clicked!"); }); // Or with arrow function button.addEventListener('click', () => { console.log("Button clicked!"); });

The Event Object

Event listeners receive an event object with useful information:

button.addEventListener('click', function(event) { console.log(event); // The event object console.log(event.target); // Element that was clicked console.log(event.type); // Type of event (click) });

Common Event Patterns

1 Click Events
let button = document.querySelector('#btn'); button.addEventListener('click', () => { alert("Button clicked!"); });
2 Input Events (typing)
let input = document.querySelector('#search'); input.addEventListener('input', () => { console.log("User typed:", input.value); });
3 Mouse Hover
let box = document.querySelector('.box'); box.addEventListener('mouseenter', () => { box.style.backgroundColor = 'yellow'; }); box.addEventListener('mouseleave', () => { box.style.backgroundColor = 'white'; });
4 Keyboard Events
document.addEventListener('keydown', (event) => { console.log("Key pressed:", event.key); if (event.key === 'Enter') { console.log("Enter key pressed!"); } });
5 Form Submit
let form = document.querySelector('#myForm'); form.addEventListener('submit', (event) => { event.preventDefault(); // Stop form from submitting console.log("Form submitted!"); // Handle form data here });
üí° event.preventDefault(): This stops the default action. Use it to:
‚Ä¢ Prevent forms from submitting and reloading the page
‚Ä¢ Prevent links from navigating
‚Ä¢ Handle the action with JavaScript instead

Multiple Event Listeners

You can add multiple listeners to the same element:

let button = document.querySelector('button'); button.addEventListener('click', () => { console.log("First listener"); }); button.addEventListener('click', () => { console.log("Second listener"); }); // Both will run when button is clicked!

Event Delegation

Listen on a parent element for events from children:

// HTML: <ul id="list"> // <li>Item 1</li> // <li>Item 2</li> // </ul> let list = document.querySelector('#list'); // Listen on the list, but check what was clicked list.addEventListener('click', (event) => { if (event.target.tagName === 'LI') { console.log("Clicked:", event.target.textContent); } });
Why use event delegation?
‚Ä¢ More efficient than adding listeners to every child
‚Ä¢ Works for elements added dynamically later
‚Ä¢ Less memory usage

Creating Elements Dynamically

JavaScript can create new HTML elements and add them to the page!

1. Creating Elements

Use createElement() to make new elements:

// Create a new paragraph let newPara = document.createElement('p'); // Set its text newPara.textContent = "This is a new paragraph!"; // Add a class newPara.classList.add('highlight'); // Add it to the page document.body.appendChild(newPara);

2. Adding Elements to the DOM

1 appendChild() - Add to end
let container = document.querySelector('.container'); let newDiv = document.createElement('div'); newDiv.textContent = "New content"; container.appendChild(newDiv); // Adds to end of container
2 insertBefore() - Add before specific element
let parent = document.querySelector('.parent'); let referenceNode = document.querySelector('.reference'); let newElement = document.createElement('div'); parent.insertBefore(newElement, referenceNode);
3 prepend() - Add to beginning (modern)
let container = document.querySelector('.container'); let newElement = document.createElement('p'); container.prepend(newElement); // Adds to beginning

3. Removing Elements

// Modern way let element = document.querySelector('.to-remove'); element.remove(); // Old way (still works) let parent = element.parentNode; parent.removeChild(element);

Let's Build a To-Do List App!

Put everything together to create a fully functional to-do list with add, delete, and complete functionality!

1 Create the HTML Structure
<!DOCTYPE html> <html> <head> <title>To-Do List</title> <style> /* CSS will go here */ </style> </head> <body> <div class="container"> <h1>My To-Do List</h1> <div class="input-section"> <input type="text" id="taskInput" placeholder="Add a new task..."> <button id="addBtn">Add</button> </div> <ul id="taskList"></ul> </div> <script> // JavaScript will go here </script> </body> </html>
2 Add CSS Styling
.container { max-width: 600px; margin: 50px auto; padding: 30px; background: white; border-radius: 10px; box-shadow: 0 5px 20px rgba(0,0,0,0.1); } .input-section { display: flex; gap: 10px; margin-bottom: 20px; } #taskInput { flex: 1; padding: 12px; border: 2px solid #ddd; border-radius: 5px; font-size: 16px; } #addBtn { padding: 12px 30px; background: #06a77d; color: white; border: none; border-radius: 5px; cursor: pointer; } #taskList { list-style: none; padding: 0; } .task-item { display: flex; align-items: center; padding: 15px; background: #f8f9fa; margin-bottom: 10px; border-radius: 5px; } .task-item.completed { opacity: 0.6; text-decoration: line-through; } .task-text { flex: 1; cursor: pointer; } .delete-btn { background: #dc3545; color: white; border: none; padding: 5px 15px; border-radius: 3px; cursor: pointer; }
3 Write the JavaScript
// Get elements const taskInput = document.querySelector('#taskInput'); const addBtn = document.querySelector('#addBtn'); const taskList = document.querySelector('#taskList'); // Add task function function addTask() { const taskText = taskInput.value.trim(); // Check if input is empty if (taskText === '') { alert('Please enter a task!'); return; } // Create task item const li = document.createElement('li'); li.className = 'task-item'; // Create task text const span = document.createElement('span'); span.className = 'task-text'; span.textContent = taskText; // Toggle complete on click span.addEventListener('click', () => { li.classList.toggle('completed'); }); // Create delete button const deleteBtn = document.createElement('button'); deleteBtn.className = 'delete-btn'; deleteBtn.textContent = 'Delete'; // Delete task on click deleteBtn.addEventListener('click', () => { li.remove(); }); // Assemble and add to list li.appendChild(span); li.appendChild(deleteBtn); taskList.appendChild(li); // Clear input taskInput.value = ''; taskInput.focus(); } // Event listeners addBtn.addEventListener('click', addTask); // Allow Enter key to add task taskInput.addEventListener('keypress', (event) => { if (event.key === 'Enter') { addTask(); } });
üí° How It Works:
1. User types a task and clicks Add (or presses Enter)
2. JavaScript creates new HTML elements dynamically
3. Click task text to mark complete (toggles class)
4. Click Delete button to remove task
5. All happens without page reload!
