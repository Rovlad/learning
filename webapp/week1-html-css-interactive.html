How Websites Work

Before we start building, let's understand what happens when you visit a website!

What is a Web Browser?

A browser (like Chrome, Firefox, Safari, or Edge) is a special program that reads code and turns it into beautiful webpages you can see and interact with.

Think of it like this:
‚Ä¢ HTML code is like a recipe written in a special language
‚Ä¢ The browser is like a chef who reads the recipe
‚Ä¢ The webpage you see is the finished meal!

When you type a website address (like google.com), the browser downloads the HTML code from the internet and instantly displays it as a webpage.

The Three Languages of the Web

Every website you've ever visited is built using three main languages working together:

  • HTML - The structure (what's on the page)
  • CSS - The style (how it looks)
  • JavaScript - The behavior (how it works) - we'll learn this later!

What is HTML?

HTML (HyperText Markup Language) is the skeleton of every webpage. It defines the structure and content using tags.

üí° Why "HyperText"? Because it can link to other pages! That's what makes the web a "web" of connected pages.

How HTML Tags Work

HTML uses tags (written with angle brackets like < >) to tell the browser what each piece of content is.

Most tags come in pairs - an opening tag and a closing tag:

<h1>This is a heading</h1> <p>This is a paragraph</p>

Notice the closing tag has a forward slash (/) before the tag name!

Basic HTML Structure

Every HTML page follows this basic structure:

<!DOCTYPE html> <!-- Tells browser this is HTML5 --> <html> <!-- Everything goes inside here --> <head> <!-- Information ABOUT the page --> <title>My First Page</title> </head> <body> <!-- What you SEE on the page --> <h1>Hello, World!</h1> <p>This is a paragraph.</p> </body> </html>
Understanding the structure:
‚Ä¢ <head> = Information the browser needs (title, styles, etc.)
‚Ä¢ <body> = Everything you actually see on the webpage
‚Ä¢ Think of <head> as backstage and <body> as the stage!

Common HTML Tags

Here are the most important tags you'll use:

  • <h1> to <h6> - Headings (h1 is biggest, h6 is smallest)
  • <p> - Paragraph (regular text)
  • <a href="url"> - Link (takes you to another page)
  • <img src="url"> - Image (shows a picture)
  • <div> - Container/box (groups things together)
  • <ul> and <li> - Unordered list (bullet points)
  • <strong> - Bold text (important!)
  • <em> - Italic text (emphasis)
üí° Self-closing tags: Some tags like <img> and <br> don't need a closing tag because they don't wrap around content!

What is CSS?

CSS (Cascading Style Sheets) makes your webpage beautiful. While HTML creates the structure, CSS controls how everything looks - colors, fonts, spacing, layout, and more.

HTML vs CSS analogy:
Imagine building a house:
‚Ä¢ HTML = The walls, doors, and windows (structure)
‚Ä¢ CSS = The paint, furniture, and decorations (style)

You need both to make something beautiful AND functional!

How CSS Works

CSS uses selectors to target HTML elements, then applies properties to style them:

selector { property: value; property: value; }

Basic CSS Syntax

Here's a real example - styling all heading 1 elements:

h1 { color: blue; font-size: 36px; text-align: center; }

This says: "Find all h1 tags and make them blue, 36 pixels tall, and centered"

Where Does CSS Go?

You put CSS inside <style> tags in the <head> section:

<head> <style> body { background-color: lightblue; } </style> </head>

Common CSS Properties

These are the properties you'll use most often:

  • color - Text color
  • background-color - Background color
  • font-size - Size of text (use px for pixels)
  • font-family - Type of font (Arial, Times New Roman, etc.)
  • padding - Space inside an element
  • margin - Space outside an element
  • border - Border around element
  • text-align - Alignment (left, center, right)
Padding vs Margin:
Think of an element like a picture frame:
‚Ä¢ Padding = Space between the picture and the frame
‚Ä¢ Margin = Space between the frame and other objects

Both create space, but in different places!
üí° Tip: CSS colors can be written as names (blue), hex codes (#0000ff), or RGB values (rgb(0,0,255)). They all work!

Try It Yourself!

The best way to learn is to experiment! Try changing values in the code editor on the right and see what happens. You can't break anything - just refresh if something goes wrong!

Let's Build Your About Me Page!

Follow these steps to create your first webpage. Type the code in the editor on the right.

1 Start with the basic structure
<!DOCTYPE html> <html> <head> <title>About Me</title> </head> <body> </body> </html>
2 Add your name as a heading

Inside the <body> tag, add:

<h1>Your Name Here</h1>
3 Add a paragraph about yourself
<p>Hi! I'm learning to build websites. I love [your hobbies here].</p>
4 Add a section about your interests
<h2>My Interests</h2> <ul> <li>Interest 1</li> <li>Interest 2</li> <li>Interest 3</li> </ul>
5 Add an image (optional)
<img src="https://via.placeholder.com/200" alt="Your description">

Replace the URL with any image URL you like!

üí° Tip: Click "Update Preview" after each step to see your changes!

Make It Beautiful with CSS!

Now let's add some style to make your page look amazing.

1 Add a style section

In the <head> section, add this:

<style> /* Your CSS will go here */ </style>
2 Style the body
body { font-family: Arial, sans-serif; background-color: #f0f8ff; padding: 20px; max-width: 800px; margin: 0 auto; }
3 Style the heading
h1 { color: #2c3e50; text-align: center; border-bottom: 3px solid #3498db; padding-bottom: 10px; }
4 Style paragraphs
p { line-height: 1.6; color: #555; }
5 Style the image
img { border-radius: 10px; display: block; margin: 20px auto; }

üåê Put Your Page on the Internet!

Let's make your page live so anyone in the world can see it! We'll use GitHub (a place where developers share code) and GitHub Pages (free website hosting).

Part 1: Create a GitHub Account

1 Go to GitHub

Open a new tab and go to github.com

2 Sign Up

Click "Sign up" and create your account:

  • Choose a username (this will be in your website URL!)
  • Use your email address
  • Create a strong password
  • Verify you're human (follow the puzzle)
3 Verify Your Email

Check your email for a verification message from GitHub and click the link inside.

üí° Tip: Choose a professional username - you might use this for years! Many developers use their real name or a creative variation.

Part 2: Create Your First Repository

4 Create a New Repository

Once logged in, click the green "New" button (or the + icon in the top right)

5 Name Your Repository

Very important! Name it exactly: yourusername.github.io

Replace "yourusername" with your actual GitHub username. For example, if your username is "sarahcodes", name it: sarahcodes.github.io

Why this name? This special name tells GitHub you want to create a website. It will automatically become your website address!
6 Repository Settings
  • Make it Public (so anyone can visit your site)
  • Check "Add a README file"
  • Click "Create repository"

Part 3: Add Your Webpage

7 Create index.html File

In your new repository, click "Add file" ‚Üí "Create new file"

Name it: index.html

8 Copy Your Code

From the editor on the right, copy ALL your HTML code (including the <!DOCTYPE html> at the top)

Paste it into the GitHub file editor

9 Commit Your Changes

Scroll down and click the green "Commit changes" button

üí° What's a commit? It's like saving your work. You're telling GitHub: "Save this version of my file!"

Part 4: See Your Live Website! üéâ

10 Wait a Moment

GitHub Pages takes about 1-2 minutes to publish your site

11 Visit Your Website!

Open a new tab and go to:

https://yourusername.github.io

Replace "yourusername" with your GitHub username

üéä Congratulations! You just put your first website on the internet! Share this link with friends and family!

Making Changes Later

12 Update Your Website

To change your website later:

  • Go to your repository on GitHub
  • Click on index.html
  • Click the pencil icon (‚úèÔ∏è) to edit

Troubleshooting

  • Seeing old content? Hold Shift and click Refresh in your browser
  • Changes not appearing? Wait a few minutes - GitHub Pages takes time to update
