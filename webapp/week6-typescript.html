What is TypeScript?

TypeScript is JavaScript with types. It's a programming language created by Microsoft that adds optional type checking to JavaScript.

Think of it like this:
‚Ä¢ JavaScript lets you put any value in any variable
‚Ä¢ TypeScript lets you say "this variable can ONLY be a number"
‚Ä¢ If you try to put text in that variable, TypeScript catches the error BEFORE you run the code!

It's like having a helpful assistant checking your work.

JavaScript vs TypeScript

‚ùå JavaScript (No Types)

let age = 25; age = "twenty-five"; // Allowed! age = true; // Also allowed! age = [1, 2, 3]; // Still allowed! // Bug found at runtime üí•

‚úÖ TypeScript (With Types)

let age: number = 25; age = "twenty-five"; // Error! ‚ùå age = true; // Error! ‚ùå age = [1, 2, 3]; // Error! ‚ùå // Bug found immediately! ‚úÖ

How TypeScript Works

  1. You write TypeScript code (.ts files)
  2. TypeScript checks your code for type errors
  3. If no errors, it compiles to regular JavaScript (.js files)
  4. The JavaScript runs in the browser

Why Types Matter

Let's see real examples of bugs that TypeScript prevents!

Example 1: Typos in Property Names

JavaScript (Bug Hidden)

let user = { name: "Sarah", age: 15 }; // Typo! Returns undefined console.log(user.agee); // No error, prints: undefined

TypeScript (Catches Bug)

let user = { name: "Sarah", age: 15 }; // Typo caught! console.log(user.agee); // Error: Property 'agee' does not // exist on type. Did you mean 'age'?

Example 2: Wrong Function Arguments

JavaScript (Runtime Error)

function greet(name) { return "Hello, " + name.toUpperCase(); } // Passes a number instead! greet(123); // Crashes: name.toUpperCase // is not a function

TypeScript (Compile Error)

function greet(name: string) { return "Hello, " + name.toUpperCase(); } // Error caught before running! greet(123); // Error: Argument of type 'number' // is not assignable to parameter // of type 'string'

Example 3: Null/Undefined Bugs

JavaScript

function getLength(str) { return str.length; } // Crashes if null! getLength(null); // Cannot read property // 'length' of null

TypeScript

function getLength(str: string) { return str.length; } // Error caught! getLength(null); // Error: Argument of type 'null' // is not assignable to parameter // of type 'string'

Example 4: Wrong Return Type

JavaScript

function add(a, b) { // Forgot to return! a + b; } let result = add(5, 3); console.log(result); // undefined (bug hidden!)

TypeScript

function add(a: number, b: number): number { // Forgot to return! a + b; } // Error: Function lacks ending // return statement and return type // does not include 'undefined'

Basic Types in TypeScript

TypeScript has several built-in types for common JavaScript values:

1. Primitive Types

1 string - Text values
let name: string = "Sarah"; let message: string = 'Hello'; let template: string = `Hi, ${name}`;
2 number - All numbers (integers and decimals)
let age: number = 15; let price: number = 19.99; let hex: number = 0xFF; // Also a number
3 boolean - True or false
let isStudent: boolean = true; let isComplete: boolean = false;

2. Array Types

// Array of strings let names: string[] = ["Alice", "Bob", "Charlie"]; // Array of numbers let scores: number[] = [85, 90, 78]; // Alternative syntax (same thing) let names: Array<string> = ["Alice", "Bob"]; // Mixed array (any type) let mixed: any[] = ["hello", 42, true];

3. Object Types

// Inline object type let user: { name: string; age: number } = { name: "Sarah", age: 15 }; // With optional properties (use ?) let person: { name: string; age?: number } = { name: "John" // age is optional, so this is valid };

4. Function Types

// Function with typed parameters and return function add(a: number, b: number): number { return a + b; } // Function that returns nothing (void) function log(message: string): void { console.log(message); } // Arrow function with types const multiply = (x: number, y: number): number => x * y;

5. Special Types

1 any - Disables type checking (avoid if possible!)
let anything: any = "hello"; anything = 42; // OK anything = true; // OK // Use sparingly - defeats purpose of TypeScript!
2 unknown - Safer than any (must check type first)
let value: unknown = "hello"; // Must check type before using if (typeof value === "string") { console.log(value.toUpperCase()); // OK now }
3 null and undefined
let nothing: null = null; let notSet: undefined = undefined; // Often used with union types (see below)

6. Union Types (Multiple Possible Types)

// Variable can be string OR number let id: string | number; id = "ABC123"; // OK id = 12345; // OK id = true; // Error! // Function parameter with union type function print(value: string | number) { console.log(value); }

7. Literal Types (Exact Values)

// Can only be these exact strings let direction: "up" | "down" | "left" | "right"; direction = "up"; // OK direction = "north"; // Error! // Useful for options type Status = "pending" | "approved" | "rejected"; let status: Status = "pending";

8. Type Inference (TypeScript Guesses!)

// TypeScript infers the type automatically let name = "Sarah"; // TypeScript knows it's a string let age = 15; // TypeScript knows it's a number // No need to write types here! function add(a: number, b: number) { return a + b; // TypeScript infers return is number }
üí° Best Practice: Let TypeScript infer types when obvious. Only add explicit types when needed for clarity or when TypeScript can't infer correctly.

Interfaces - Defining Object Shapes

Interfaces let you define the structure of objects. Think of them as contracts that say "this object must have these properties."

Basic Interface

// Define the shape of a User interface User { name: string; age: number; email: string; } // Use the interface let user: User = { name: "Sarah", age: 15, email: "sarah@example.com" }; // Error: missing 'email' property let incomplete: User = { name: "John", age: 20 };

Optional Properties

interface Person { name: string; age: number; phone?: string; // Optional (notice the ?) } // Valid - phone is optional let person1: Person = { name: "Alice", age: 25 }; // Also valid - can include phone let person2: Person = { name: "Bob", age: 30, phone: "555-1234" };

Readonly Properties

interface Book { readonly isbn: string; // Can't be changed after creation title: string; author: string; } let book: Book = { isbn: "978-0-123456-78-9", title: "TypeScript Basics", author: "Jane Doe" }; book.title = "New Title"; // OK book.isbn = "new-isbn"; // Error! readonly property

Interface for Functions

interface MathOperation { (a: number, b: number): number; } // Functions that match the interface const add: MathOperation = (x, y) => x + y; const subtract: MathOperation = (x, y) => x - y; const multiply: MathOperation = (x, y) => x * y;

Interface for Arrays

interface Task { id: number; text: string; completed: boolean; } // Array of Task objects let tasks: Task[] = [ { id: 1, text: "Learn TypeScript", completed: false }, { id: 2, text: "Build a project", completed: false } ];

Extending Interfaces

// Base interface interface Animal { name: string; age: number; } // Extend with more properties interface Dog extends Animal { breed: string; bark(): void; } let myDog: Dog = { name: "Max", age: 3, breed: "Golden Retriever", bark() { console.log("Woof!"); } };

Real Example: To-Do App

// Define Task interface interface Task { id: number; text: string; completed: boolean; createdAt: Date; priority?: "low" | "medium" | "high"; } // Function to add task function addTask(text: string, tasks: Task[]): Task { const newTask: Task = { id: Date.now(), text: text, completed: false, createdAt: new Date() }; tasks.push(newTask); return newTask; } // Function to toggle task function toggleTask(task: Task): void { task.completed = !task.completed; } // TypeScript ensures we use tasks correctly! let myTasks: Task[] = []; addTask("Learn TypeScript", myTasks);
üí° Type vs Interface: TypeScript also has type aliases which are similar to interfaces. For object shapes, either works, but interface is more common and can be extended more easily.

Build: TypeScript To-Do List

Let's build a complete typed to-do list application!

1 Define Types and Interfaces
// types.ts - Type definitions interface Task { id: number; text: string; completed: boolean; createdAt: Date; priority?: "low" | "medium" | "high"; } type TaskFilter = "all" | "active" | "completed";
2 Storage Functions with Types
// storage.ts - Typed storage operations function loadTasks(): Task[] { const data = localStorage.getItem('tasks'); if (!data) return []; const tasks = JSON.parse(data) as Task[]; // Convert date strings back to Date objects return tasks.map(task => ({ ...task, createdAt: new Date(task.createdAt) })); } function saveTasks(tasks: Task[]): void { localStorage.setItem('tasks', JSON.stringify(tasks)); }
3 Task Management Functions
// taskManager.ts - Core logic with types let tasks: Task[] = loadTasks(); let currentFilter: TaskFilter = "all"; function addTask(text: string, priority?: "low" | "medium" | "high"): Task { const newTask: Task = { id: Date.now(), text: text.trim(), completed: false, createdAt: new Date(), priority: priority }; tasks.push(newTask); saveTasks(tasks); return newTask; } function toggleTask(id: number): void { const task = tasks.find(t => t.id === id); if (task) { task.completed = !task.completed; saveTasks(tasks); } } function deleteTask(id: number): void { tasks = tasks.filter(t => t.id !== id); saveTasks(tasks); } function getFilteredTasks(): Task[] { switch (currentFilter) { case "active": return tasks.filter(t => !t.completed); case "completed": return tasks.filter(t => t.completed); default: return tasks; } } function setFilter(filter: TaskFilter): void { currentFilter = filter; }
4 DOM Functions with Types
// ui.ts - UI functions with DOM types function renderTasks(): void { const taskList = document.querySelector('#taskList') as HTMLUListElement; const filteredTasks = getFilteredTasks(); taskList.innerHTML = ''; filteredTasks.forEach((task: Task) => { const li = document.createElement('li'); li.className = 'task-item'; if (task.completed) { li.classList.add('completed'); } const span = document.createElement('span'); span.textContent = task.text; span.addEventListener('click', () => { toggleTask(task.id); renderTasks(); }); const deleteBtn = document.createElement('button'); deleteBtn.textContent = 'Delete'; deleteBtn.addEventListener('click', () => { deleteTask(task.id); renderTasks(); }); li.appendChild(span); li.appendChild(deleteBtn); taskList.appendChild(li); }); updateStats(); } function updateStats(): void { const total = tasks.length; const completed = tasks.filter(t => t.completed).length; const statsEl = document.querySelector('#stats') as HTMLElement; statsEl.textContent = `${total} tasks (${completed} completed)`; }
5 Initialize App
// main.ts - App initialization function init(): void { const addBtn = document.querySelector('#addBtn') as HTMLButtonElement; const input = document.querySelector('#taskInput') as HTMLInputElement; addBtn.addEventListener('click', () => { if (input.value.trim()) { addTask(input.value); input.value = ''; renderTasks(); } }); input.addEventListener('keypress', (e: KeyboardEvent) => { if (e.key === 'Enter') { addBtn.click(); } }); renderTasks(); } // Start app when DOM is ready document.addEventListener('DOMContentLoaded', init);

Without Types (JavaScript)

function calculateTotal(items) { let total = 0; for (let item of items) { total += item.price * item.quantity; } return total; } // What if someone passes wrong data? calculateTotal("not an array"); // Crashes! calculateTotal([{ pric: 10 }]); // Typo! Returns NaN

With Types (TypeScript)

interface CartItem { name: string; price: number; quantity: number; } function calculateTotal(items: CartItem[]): number { let total = 0; for (let item of items) { total += item.price * item.quantity; } return total; } // TypeScript catches these errors immediately! calculateTotal("not an array"); // Error: Type 'string' is not... calculateTotal([{ pric: 10 }]); // Error: Property 'price' is missing // Only correct usage compiles calculateTotal([ { name: "Book", price: 20, quantity: 2 } ]); // ‚úÖ Works!
